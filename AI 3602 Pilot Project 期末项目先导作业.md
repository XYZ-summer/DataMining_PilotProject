# AI 3602 Pilot Project 期末项目先导作业

## 概览
本项目为期末课程项目的先导作业。项目内容是：运用数据挖掘的方法，结合现代 AI 编程工具，在公开的、相对原始的互联网数据集中，挖掘和清洗 **特定选题方向** 的数据。

### 选题三：基于 GAKG 知识图谱的 Acemap 搜索增强

#### 背景

GAKG 是一个在地学文献上构建的知识图谱。其中包括从地学文献中自动抽取的 `(subject, relation, object)` 三元组。

[Acemap](https://acemap.info/) 则是一个在更多领域的海量学术资源上构建的数据库。除了数据库之外，Acemap 主页提供了一个配套的搜索引擎。目前，Acemap 中的文献检索功能基于 ElasticSearch 和关系型数据库。由于 ElasticSearch 的默认检索功能主要基于关键词匹配，返回的结果有时可能不够全面，有时则包含与用户搜索意图无关的内容。

我们考虑通过知识图谱对 Acemap 的搜索功能进行增强。Acemap 支持搜索关键词、作者、机构等。为了简化任务，我们只考虑 **地学领域的关键词搜索**。搜索的工作流程是：

1. 用户输入关键词
2. 根据关键词，返回若干篇匹配的文献/学术概念

#### 选题目标

本项目的目标是：在 Acemap 现有的搜索系统上，基于知识图谱，增加一个 “预处理” 或 “后处理” 过程，使 Acemap 搜索功能返回的结果更加准确、丰富、完善。

可以参考的思路包括但不限于

- **从提高搜索精度的角度**：利用知识图谱对搜索结果进行筛选（例如，如果某篇文章的关键词不在用户搜索关键词的临近节点内，则降低该条目的排名或将其移除）。
- **从召回相关知识的角度**：引入知识图谱，在正式搜索前，用知识图谱找到用户输入的关键词的 **相关概念**，然后调用搜索接口，返回更多相关文献。
- **从增强搜索结果的角度**：在搜索结束后，用首次搜索的结果，结合知识图谱（或其他搜索接口）对结果进行增强，返回“多跳搜索”的相关工作（例如相同作者、相同机构、相同合作者的相关领域工作）。
- **搜索意图理解**：ElasticSearch 的搜索以关键词匹配为主，因此无法处理类似 "recent research results on plate tectonics" 的非关键词查询。可尝试结合知识图谱与意图理解模型，处理用户的搜索意图。
- **其他思路**：本先导项目以完成性目标为主，我们鼓励同学们结合可用的数据，探索各类方案。

#### 数据来源

- Acemap 的主要搜索接口：
    - 论文搜索：`https://acemap.info/api/v1/work/search`
        - 样例：`https://acemap.info/api/v1/work/search?keyword=rock&order=desc&page=1&size=10`，搜索关键词 `rock`，每页 10 个结果，返回第 1 页，按相关性降序排序。
        - 可将该 URL 复制到浏览器地址栏获取 Response。
    - 作者搜索：`https://acemap.info/api/v1/author/search`
        - 样例：`https://acemap.info/api/v1/author/search?keyword=john&page=1&size=10`，搜索关键词 `john`，每页 10 个结果，返回第 1 页。
    - 机构搜索：`https://acemap.info/api/v1/institution/search`
        - 样例：`https://acemap.info/api/v1/institution/search?keyword=jiaotong%20university&page=1&size=10`，搜索关键词 `jiaotong university`，每页 10 个结果，返回第 1 页。
    - 实际开发时，可以使用 Python 的 `request` （或其他类似的库）发送 HTTP 请求。
- GAKG 知识图谱：我们在课程的 Canvas 上提供了从 GAKG 中抽取的子图（[链接](https://oc.sjtu.edu.cn/courses/81842/files/folder/%E5%85%88%E5%AF%BC%E9%A1%B9%E7%9B%AE%E4%B8%89-GAKG%E5%AD%90%E5%9B%BE)）。包含 `gakg-subset.parquet` 一张表。
    - `gakg-subset` 是 GAKG 中所有关系三元组的子集，其中包含四个字段：`subject`, `relation`, `object`, `paperid`，对应关系三元组的主谓宾，以及该三元组的来源论文 ID。
    - 如果后续课程项目需要更多 GAKG 的数据或相关信息，可以与助教联系。
- [ConceptNet](https://conceptnet.io/) 提供了更广泛、全面的知识图谱。
    - 可以针对特定领域，从 ConceptNet 抽取子图，作为搜索时的知识增强。
- 也可以搜集其他特定领域的、开源的知识图谱。

由于 GAKG 是地学领域的知识图谱，我们建议只考虑地学领域的搜索能力提升。但是，如果选用其他知识图谱，也可以考虑其他领域（只需考虑某个学科或研究领域即可）。

#### 评估指标（完成性指标）

- 功能完备性：一个可复现的代码或脚本。在现有搜索 API 的基础上，**完成至少一项增强设计，且搜索常见关键词时不报错，即视为满分**（先导项目以功能实现为主，不设额外的性能或指标要求）。
- 数据收集：与搜索功能配套的数据或增强功能（知识图谱子图、作者机构信息等）
- 可复现性：代码复现结果与汇报结果基本一致
- 其余拓展功能或性能指标不计入先导项目评分，可留作期末课程项目内容。

#### 后续目标（供期末项目参考）

- **对 Acemap 搜索功能的深入完善**：在先导项目的基础上，结合知识图谱和/或大模型，进一步优化搜索效果。
- **知识图谱的清洗和扩充**：项目提供的 GAKG 知识图谱中可能包含冗余或噪声。后续可以探讨如何利用图数据挖掘算法，清除其中的噪声。
- **在 GAKG 上探索图数据算法的应用**：尝试将课程中涉及的（或其他相关的）图数据挖掘算法应用到 GAKG 数据集上，并以此为基础探索 GAKG 的可能应用。
- **\*基于知识图谱和的 RAG**：根据知识图谱和/或搜索结果，搭建搜索增强生成（RAG）系统
- **\*个性化搜索**：根据用户的搜索记录和用户画像，返回个性化的论文搜索推荐。
- **其他**。
